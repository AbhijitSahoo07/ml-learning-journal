# Self-Attention

## Overview
Self-Attention is a powerful mechanism that allows a neural network to weigh the importance of different parts of an input sequence when processing a specific element in that sequence. Think of it like this: when you read a sentence, to understand a particular word, you don't just look at that word in isolation. You look at the surrounding words, and some words are more relevant to understanding the current word than others. Self-Attention mimics this human ability to focus on relevant parts of the input.

It's a core component of the Transformer architecture, which revolutionized Natural Language Processing (NLP) and has since expanded its influence to computer vision and other domains. Unlike traditional recurrent neural networks (RNNs) that process sequences word by word, Self-Attention can process all words in a sequence simultaneously, allowing it to capture long-range dependencies much more effectively and efficiently.

In essence, Self-Attention computes a "contextualized" representation for each element in a sequence by aggregating information from all other elements, weighted by their relevance to the current element. This dynamic weighting is what makes it so effective at understanding context.

## What Problem It Solves
Self-Attention primarily addresses several limitations inherent in previous neural network architectures, especially when dealing with sequential data like text:

1.  **Capturing Long-Range Dependencies (The Vanishing Gradient Problem in RNNs):**
    *   Traditional Recurrent Neural Networks (RNNs) and their variants (LSTMs, GRUs) process sequences one element at a time. To understand a word at the end of a long sentence, an RNN needs to carry information from words at the beginning through many steps.
    *   This sequential processing often leads to the "vanishing gradient problem," where gradients become too small to effectively update weights for earlier parts of the sequence, making it difficult to learn dependencies between distant words.
    *   Self-Attention allows each word to directly "look at" and weigh every other word in the sequence, regardless of their distance, making it excellent at capturing these long-range dependencies.

2.  **Parallelization and Computational Efficiency:**
    *   RNNs are inherently sequential, meaning they cannot be easily parallelized during training. Each step depends on the output of the previous step. This makes them slow to train on very long sequences.
    *   Self-Attention, on the other hand, can compute the attention weights for all elements in parallel. This significantly speeds up training, especially on modern hardware like GPUs, which are optimized for parallel computations.

3.  **Fixed-Size Context Window (in CNNs for sequences):**
    *   Convolutional Neural Networks (CNNs) can also be used for sequence processing, but they typically operate with a fixed-size "receptive field" or kernel. To capture long-range dependencies, one would need many layers of convolutions or very large kernels, which can be computationally expensive and less flexible.
    *   Self-Attention provides a dynamic, adaptive "receptive field" where the context for each element is determined by its learned relevance to all other elements, not a fixed window size.

4.  **Lack of Contextual Understanding:**
    *   Word embeddings (like Word2Vec or GloVe) provide a static representation for each word, meaning the word "bank" has the same embedding whether it refers to a financial institution or a river bank.
    *   Self-Attention allows the model to create *contextualized* embeddings. The representation of "bank" will be different depending on the other words in the sentence (e.g., "river bank" vs. "money bank"), as it dynamically weighs the influence of surrounding words.

In summary, Self-Attention provides a more efficient, flexible, and powerful way to model relationships within a sequence, leading to significant advancements in tasks requiring deep contextual understanding.

## How It Works
Let's break down the mechanism of Self-Attention step-by-step. Imagine you have a sequence of words, and each word is represented by an embedding vector.

1.  **Input Embeddings:**
    *   Start with your input sequence, where each item (e.g., a word in a sentence) is represented as a vector (an embedding). Let's say we have a sequence of $N$ words, and each word is represented by a vector of dimension $d_{model}$.

2.  **Generate Query, Key, and Value Vectors:**
    *   For each input embedding, we create three new vectors: a **Query** vector ($Q$), a **Key** vector ($K$), and a **Value** vector ($V$).
    *   These are generated by multiplying the input embedding vector by three different learned weight matrices: $W^Q$, $W^K$, and $W^V$. These weight matrices are unique for each attention head (if using Multi-Head Attention) and are learned during training.
    *   The dimensions of $Q$, $K$, and $V$ are typically smaller than $d_{model}$, often $d_k$ for Query/Key and $d_v$ for Value. For simplicity, often $d_k = d_v$.
    *   **Intuition:**
        *   **Query (Q):** Represents "what I'm looking for" or "what information I need from other words."
        *   **Key (K):** Represents "what information I can offer" or "what content I have."
        *   **Value (V):** Represents the actual content or information to be aggregated if selected.

3.  **Calculate Attention Scores:**
    *   For each word's Query vector, we calculate a "score" against every other word's Key vector (including itself). This score determines how much attention the current word should pay to other words.
    *   The most common way to calculate this score is using the **dot product** between the Query vector of the current word and the Key vector of another word. A higher dot product means higher similarity and thus higher relevance.

4.  **Scale the Scores:**
    *   The dot product scores can become very large, especially with high-dimensional key vectors, which can push the softmax function into regions with very small gradients, making training unstable.
    *   To counteract this, the scores are typically divided by the square root of the dimension of the Key vectors, $\sqrt{d_k}$. This scaling helps stabilize the gradients during training.

5.  **Apply Softmax:**
    *   The scaled scores are then passed through a **softmax** function. This converts the scores into probabilities, ensuring they are all positive and sum up to 1. These probabilities represent the attention weights – how much each word should "attend" to every other word.

6.  **Compute Weighted Sum of Value Vectors:**
    *   Finally, for each word, its new, contextualized representation is created by taking a weighted sum of all the Value vectors in the sequence. The weights are the attention probabilities calculated in the previous step.
    *   This means that words with higher attention weights (more relevant) contribute more to the output representation of the current word.

7.  **Output:**
    *   The result is a new set of vectors, one for each input word, where each vector now incorporates information from the entire sequence, weighted by relevance. These are the "contextualized embeddings."

**In summary, for each word:**
1.  It asks "What information do I need?" (Query).
2.  It compares its Query to "What information do other words offer?" (Keys).
3.  It gets a relevance score for each other word.
4.  It normalizes these scores (softmax).
5.  It then sums up the "content" (Values) of all other words, weighted by these relevance scores, to form its new, context-aware representation.

This entire process is often done in parallel for all words in the sequence, making it highly efficient.

## Mathematical Intuition
Let's formalize the steps described above.

Assume we have an input sequence represented as a matrix $X$, where each row is an embedding vector for a word. If we have $N$ words and each embedding has dimension $d_{model}$, then $X$ is an $N \times d_{model}$ matrix.

1.  **Linear Transformations to Query, Key, Value:**
    For each input vector $x_i$ (a row in $X$), we transform it into a Query ($q_i$), Key ($k_i$), and Value ($v_i$) vector. This is done by multiplying $X$ by learned weight matrices $W^Q$, $W^K$, and $W^V$.
    Let $W^Q \in \mathbb{R}^{d_{model} \times d_k}$, $W^K \in \mathbb{R}^{d_{model} \times d_k}$, and $W^V \in \mathbb{R}^{d_{model} \times d_v}$.
    Then, the matrices for all Queries, Keys, and Values for the entire sequence are:
    $$Q = XW^Q$$
    $$K = XW^K$$
    $$V = XW^V$$
    Here, $Q$ is an $N \times d_k$ matrix, $K$ is an $N \times d_k$ matrix, and $V$ is an $N \times d_v$ matrix. Each row in $Q$, $K$, or $V$ corresponds to the Query, Key, or Value vector for a specific word in the input sequence.

2.  **Calculate Attention Scores (Dot Product):**
    To find out how much each Query vector $q_i$ should attend to each Key vector $k_j$, we compute their dot product. For all pairs of Query and Key vectors in the sequence, this can be efficiently computed as a matrix multiplication:
    $$Scores = QK^T$$
    Here, $K^T$ is the transpose of $K$, so it's a $d_k \times N$ matrix. The result $Scores$ is an $N \times N$ matrix, where $Scores_{ij}$ represents the attention score of the $i$-th query with the $j$-th key.

3.  **Scale the Scores:**
    To prevent large dot products from dominating the softmax function, we scale the scores by dividing by the square root of the dimension of the Key vectors, $\sqrt{d_k}$.
    $$ScaledScores = \frac{QK^T}{\sqrt{d_k}}$$

4.  **Apply Softmax:**
    The scaled scores are then passed through a row-wise softmax function to obtain the attention weights. Each row of the resulting matrix $AttentionWeights$ will sum to 1.
    $$AttentionWeights = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)$$
    This $AttentionWeights$ matrix is also $N \times N$. Each row $i$ of this matrix contains the attention probabilities for the $i$-th word, indicating how much it attends to every other word (column $j$).

5.  **Compute Weighted Sum of Value Vectors:**
    Finally, we multiply the $AttentionWeights$ matrix by the $V$ matrix. This performs a weighted sum of the Value vectors for each Query.
    $$Output = AttentionWeights \cdot V$$
    The $Output$ matrix is $N \times d_v$. Each row in $Output$ is the new, contextualized representation for the corresponding word in the input sequence.

Combining these steps, the full Self-Attention formula (often called Scaled Dot-Product Attention) is:
$$ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

**Explanation of terms:**
*   $Q, K, V$: Matrices of Query, Key, and Value vectors, respectively, derived from the input embeddings.
*   $Q K^T$: Matrix multiplication representing the dot product similarity between each query and all keys.
*   $d_k$: The dimension of the Key vectors. Dividing by $\sqrt{d_k}$ is a crucial scaling factor that helps stabilize training by preventing the dot products from becoming too large, especially when $d_k$ is high. Without this scaling, the softmax function could have extremely small gradients, hindering learning.
*   $\text{softmax}(\cdot)$: A function that converts a vector of real numbers into a probability distribution, where all elements are between 0 and 1 and sum to 1. Applied row-wise to the $QK^T/\sqrt{d_k}$ matrix.
*   $V$: The matrix of Value vectors. The attention weights are used to take a weighted sum of these values.

This entire process is often repeated multiple times in parallel, known as **Multi-Head Attention**, where each "head" learns different $W^Q, W^K, W^V$ matrices, allowing the model to attend to different aspects of the input simultaneously. The outputs from these multiple heads are then concatenated and linearly transformed to produce the final output.

## Advantages
Self-Attention offers several significant advantages:

*   **Captures Long-Range Dependencies:** It can directly model relationships between any two positions in a sequence, regardless of their distance, overcoming the limitations of RNNs with vanishing gradients.
*   **Parallelization:** Unlike sequential RNNs, Self-Attention computations can be performed in parallel for all elements in a sequence, leading to much faster training times on modern hardware (GPUs, TPUs).
*   **Contextual Understanding:** It generates dynamic, context-aware representations for each element, where the meaning of a word can change based on its surrounding words in the current input.
*   **Interpretability (to some extent):** The attention weights can sometimes be visualized to understand which parts of the input the model is focusing on when making a prediction, offering a degree of interpretability.
*   **Handles Variable-Length Inputs:** It can naturally process sequences of varying lengths without requiring padding to a fixed size (though padding is often used for batching).
*   **Reduced Sequential Bottleneck:** By allowing direct connections between all elements, it avoids the information bottleneck that can occur when all information must pass through a single hidden state in an RNN.

## Disadvantages
Despite its power, Self-Attention also has some drawbacks:

*   **Computational Complexity:** The core attention mechanism involves computing dot products between all pairs of Query and Key vectors. This results in a computational complexity of $O(N^2 \cdot d_k)$ where $N$ is the sequence length. For very long sequences, this quadratic complexity can become computationally expensive and memory-intensive.
*   **Memory Usage:** Storing the $N \times N$ attention matrix requires $O(N^2)$ memory, which can be a bottleneck for extremely long sequences.
*   **Lack of Positional Information:** The attention mechanism itself is permutation-invariant; it doesn't inherently know the order of words in a sequence. To address this, **Positional Encodings** must be added to the input embeddings, which adds an extra layer of complexity.
*   **Requires Large Datasets:** Transformers, which heavily rely on Self-Attention, often require very large datasets to train effectively due to their high number of parameters.
*   **Less Inductive Bias for Sequential Data:** Unlike RNNs, which have a strong inductive bias for sequential processing, Self-Attention needs to learn all sequential relationships from scratch, which is why positional encodings are crucial.

## Real World Applications
Self-Attention, particularly as part of the Transformer architecture, has become ubiquitous in many advanced AI applications:

1.  **Natural Language Processing (NLP):** This is where Self-Attention first gained prominence.
    *   **Machine Translation:** Models like Google Translate use Transformers to translate text between languages, leveraging Self-Attention to understand context in both source and target sentences.
    *   **Text Summarization:** Generating concise summaries of longer documents.
    *   **Question Answering:** Understanding a question and a given text to extract or generate an answer (e.g., BERT, GPT-3).
    *   **Text Generation:** Creating human-like text for chatbots, content creation, or code generation (e.g., GPT series, LaMDA).
    *   **Sentiment Analysis, Named Entity Recognition, Part-of-Speech Tagging:** All benefit from the deep contextual understanding provided by Self-Attention.

2.  **Computer Vision (CV):** While initially dominant in NLP, Self-Attention has made significant inroads into CV.
    *   **Vision Transformers (ViT):** These models treat image patches as sequences and apply the Transformer architecture (with Self-Attention) to achieve state-of-the-art results in image classification, object detection, and segmentation, often outperforming traditional CNNs.
    *   **Image Generation:** Models like DALL-E and Stable Diffusion use attention mechanisms to understand prompts and generate high-quality images.

3.  **Speech Recognition:**
    *   **Automatic Speech Recognition (ASR):** Transformers with Self-Attention are used to process audio sequences, converting spoken language into text. They can better handle long utterances and contextual cues in speech.

4.  **Drug Discovery and Bioinformatics:**
    *   **Protein Folding Prediction:** Models like AlphaFold use attention mechanisms to predict the 3D structure of proteins from their amino acid sequences, a critical problem in biology and drug design. Self-Attention helps model interactions between distant amino acids.
    *   **Molecular Property Prediction:** Predicting properties of molecules based on their graph structure, where attention can identify important atomic interactions.

5.  **Time Series Forecasting:**
    *   Self-Attention can be applied to time series data to identify important temporal dependencies and patterns, improving forecasting accuracy for things like stock prices, weather, or energy consumption.

## Python Example
Here's a complete, standalone Python example demonstrating the core Self-Attention mechanism using NumPy. We'll simulate a sequence of word embeddings and compute their contextualized representations.

```python
import numpy as np

def softmax(x):
    """Compute softmax values for each row of x."""
    e_x = np.exp(x - np.max(x, axis=-1, keepdims=True)) # Subtract max for numerical stability
    return e_x / e_x.sum(axis=-1, keepdims=True)

def self_attention(input_embeddings, W_q, W_k, W_v):
    """
    Implements the scaled dot-product self-attention mechanism.

    Args:
        input_embeddings (np.ndarray): Input sequence embeddings.
                                       Shape: (sequence_length, d_model)
        W_q (np.ndarray): Weight matrix for Query. Shape: (d_model, d_k)
        W_k (np.ndarray): Weight matrix for Key. Shape: (d_model, d_k)
        W_v (np.ndarray): Weight matrix for Value. Shape: (d_model, d_v)

    Returns:
        np.ndarray: Output contextualized embeddings.
                    Shape: (sequence_length, d_v)
    """
    seq_len, d_model = input_embeddings.shape
    _, d_k = W_q.shape # d_k is the dimension of Query/Key vectors
    _, d_v = W_v.shape # d_v is the dimension of Value vectors

    print(f"Input sequence length: {seq_len}, embedding dimension (d_model): {d_model}")
    print(f"Query/Key dimension (d_k): {d_k}, Value dimension (d_v): {d_v}")

    # 1. Generate Query, Key, Value matrices
    # Q = XW^Q
    # K = XW^K
    # V = XW^V
    Q = np.dot(input_embeddings, W_q) # Shape: (seq_len, d_k)
    K = np.dot(input_embeddings, W_k) # Shape: (seq_len, d_k)
    V = np.dot(input_embeddings, W_v) # Shape: (seq_len, d_v)

    print("\n--- Step 1: Generate Q, K, V matrices ---")
    print("Q (Queries):\n", Q)
    print("K (Keys):\n", K)
    print("V (Values):\n", V)

    # 2. Calculate Attention Scores (QK^T)
    # QK^T = (seq_len, d_k) @ (d_k, seq_len) = (seq_len, seq_len)
    attention_scores = np.dot(Q, K.T)

    print("\n--- Step 2: Calculate Attention Scores (QK^T) ---")
    print("Attention Scores (raw):\n", attention_scores)

    # 3. Scale the Scores
    # Divide by sqrt(d_k)
    scaling_factor = np.sqrt(d_k)
    scaled_attention_scores = attention_scores / scaling_factor

    print("\n--- Step 3: Scale the Attention Scores ---")
    print(f"Scaling factor (sqrt(d_k)): {scaling_factor:.2f}")
    print("Scaled Attention Scores:\n", scaled_attention_scores)

    # 4. Apply Softmax to get Attention Weights
    # Each row sums to 1
    attention_weights = softmax(scaled_attention_scores)

    print("\n--- Step 4: Apply Softmax to get Attention Weights ---")
    print("Attention Weights (probabilities):\n", attention_weights)
    print("Sum of each row of attention weights (should be ~1):\n", np.sum(attention_weights, axis=-1))

    # 5. Compute Weighted Sum of Value Vectors
    # Output = AttentionWeights @ V
    # (seq_len, seq_len) @ (seq_len, d_v) = (seq_len, d_v)
    output_embeddings = np.dot(attention_weights, V)

    print("\n--- Step 5: Compute Weighted Sum of Value Vectors ---")
    print("Output Contextualized Embeddings:\n", output_embeddings)

    return output_embeddings

# --- Dummy Data and Parameters ---
# Sequence length (e.g., number of words in a sentence)
sequence_length = 4
# Embedding dimension (d_model)
d_model = 8
# Dimension of Query/Key vectors (d_k)
d_k = 4
# Dimension of Value vectors (d_v)
d_v = 4 # Often d_k == d_v

# Generate dummy input embeddings (e.g., from a word embedding layer)
# Each row is an embedding for a word
np.random.seed(42) # for reproducibility
input_embeddings = np.random.rand(sequence_length, d_model)
print("Original Input Embeddings:\n", input_embeddings)

# Generate dummy weight matrices for Q, K, V
# These would be learned during training in a real model
W_q = np.random.rand(d_model, d_k)
W_k = np.random.rand(d_model, d_k)
W_v = np.random.rand(d_model, d_v)

print("\n--- Weight Matrices (Learned Parameters) ---")
print("W_q (Query weights):\n", W_q)
print("W_k (Key weights):\n", W_k)
print("W_v (Value weights):\n", W_v)

# --- Run Self-Attention ---
print("\n=====================================")
print("       Running Self-Attention        ")
print("=====================================")
contextualized_output = self_attention(input_embeddings, W_q, W_k, W_v)

print("\n=====================================")
print("           Final Output              ")
print("=====================================")
print("Original Input Embeddings (first word):\n", input_embeddings[0])
print("Contextualized Output Embedding (first word):\n", contextualized_output[0])

# Observe how the output embeddings are different from input,
# as they now incorporate information from the entire sequence.
```

**Explanation of the Code:**

1.  **`softmax(x)` function:** A helper function to compute the softmax activation. It's crucial for converting raw scores into probability distributions (attention weights). It includes a numerical stability trick by subtracting the maximum value before exponentiation.
2.  **`self_attention(input_embeddings, W_q, W_k, W_v)` function:**
    *   Takes `input_embeddings` (your sequence of word vectors) and the three weight matrices (`W_q`, `W_k`, `W_v`) as input.
    *   **Step 1: Generate Q, K, V:** It performs matrix multiplications (`np.dot`) to project the `input_embeddings` into Query, Key, and Value spaces using the respective weight matrices.
    *   **Step 2: Calculate Attention Scores:** It computes the dot product between `Q` and the transpose of `K` (`K.T`). This results in a matrix where each element `(i, j)` indicates how much the `i`-th query attends to the `j`-th key.
    *   **Step 3: Scale the Scores:** Divides the attention scores by `sqrt(d_k)` to prevent the softmax from having extremely small gradients.
    *   **Step 4: Apply Softmax:** Applies the `softmax` function row-wise to the scaled scores, turning them into attention probabilities (weights). Each row sums to 1.
    *   **Step 5: Compute Weighted Sum of Values:** Multiplies the `attention_weights` matrix by the `V` (Value) matrix. This operation effectively takes a weighted sum of all Value vectors for each output position, using the calculated attention weights.
    *   Returns the `output_embeddings`, which are the contextualized representations of the input sequence.
3.  **Dummy Data and Parameters:**
    *   We define `sequence_length`, `d_model` (embedding dimension), `d_k` (query/key dimension), and `d_v` (value dimension).
    *   `input_embeddings` is a randomly generated matrix representing our sequence.
    *   `W_q`, `W_k`, `W_v` are randomly initialized weight matrices. In a real neural network, these would be trainable parameters.
4.  **Execution:** The code then calls `self_attention` with the dummy data and prints intermediate steps to illustrate the process. You can observe how the `output_embeddings` are different from the `input_embeddings`, having been transformed by the attention mechanism.

This example demonstrates the core mathematical operations of Self-Attention. In a full Transformer model, this would be wrapped in a layer, often combined with Multi-Head Attention and other components like feed-forward networks and residual connections.

## Interview Questions

1.  **What is Self-Attention and why is it important in modern deep learning?**
    *   **Answer:** Self-Attention is a mechanism that allows a neural network to weigh the importance of different parts of an input sequence when processing a specific element in that sequence. It's crucial because it enables models to capture long-range dependencies in data (like text or images) more effectively and efficiently than previous architectures (like RNNs), leading to better contextual understanding and parallelization during training.

2.  **Explain the role of Query, Key, and Value vectors in Self-Attention.**
    *   **Answer:** For each element in the input sequence, three vectors are derived:
        *   **Query (Q):** Represents "what I'm looking for" or "what information I need."
        *   **Key (K):** Represents "what information I can offer" or "what content I have."
        *   **Value (V):** Represents the actual content or information that will be aggregated if selected.
        The Query of one element is compared against the Keys of all other elements (including itself) to determine attention scores, which then weight the corresponding Value vectors to form the output.

3.  **How does Self-Attention address the limitations of Recurrent Neural Networks (RNNs)?**
    *   **Answer:** Self-Attention addresses RNN limitations primarily in two ways:
        1.  **Long-Range Dependencies:** RNNs struggle with vanishing/exploding gradients over long sequences, making it hard to learn dependencies between distant elements. Self-Attention directly connects every element to every other element, allowing it to capture these dependencies regardless of distance.
        2.  **Parallelization:** RNNs are inherently sequential, limiting parallel computation. Self-Attention can compute attention weights for all elements simultaneously, significantly speeding up training on parallel hardware.

4.  **What is the purpose of scaling the dot product by $\sqrt{d_k}$ in Self-Attention?**
    *   **Answer:** The dot product of Query and Key vectors can grow very large, especially with high-dimensional $d_k$. When these large values are passed into the softmax function, they can push the softmax into regions where its gradient is extremely small (i.e., very steep or flat regions), leading to vanishing gradients and unstable training. Dividing by $\sqrt{d_k}$ helps to normalize these scores, keeping them in a more stable range for the softmax function, thus improving training stability and performance.

5.  **What is Multi-Head Attention and why is it used?**
    *   **Answer:** Multi-Head Attention is an extension of Self-Attention where the attention mechanism is run multiple times in parallel. Instead of performing one attention calculation, the input is linearly projected into multiple "heads" (sets of Q, K, V matrices). Each head learns different linear transformations and can focus on different aspects or relationships within the sequence. The outputs from these multiple heads are then concatenated and linearly transformed to produce the final output. It allows the model to jointly attend to information from different representation subspaces at different positions, enriching the model's ability to capture diverse relationships.

6.  **Why do Transformers need Positional Encodings when using Self-Attention?**
    *   **Answer:** Self-Attention is inherently permutation-invariant, meaning it treats a sequence as a "bag of words" and doesn't inherently know the order or position of elements. For tasks like language understanding, word order is crucial. Positional Encodings are added to the input embeddings to inject information about the relative or absolute position of each element in the sequence. This allows the model to leverage sequential information while retaining the parallelization benefits of Self-Attention.

7.  **Discuss the computational complexity of Self-Attention.**
    *   **Answer:** The dominant computational cost in Self-Attention comes from calculating the attention scores ($QK^T$) and then multiplying by the Value matrix. If $N$ is the sequence length and $d_k$ is the dimension of Query/Key vectors, calculating $QK^T$ involves an $N \times d_k$ matrix multiplied by a $d_k \times N$ matrix, resulting in an $N \times N$ matrix. This operation has a complexity of $O(N^2 \cdot d_k)$. The subsequent multiplication with $V$ is $O(N^2 \cdot d_v)$. Therefore, the overall computational complexity is quadratic with respect to the sequence length, $O(N^2)$.

8.  **Can Self-Attention be used in Computer Vision? If so, how?**
    *   **Answer:** Yes, absolutely. Self-Attention has become very prominent in Computer Vision with the advent of Vision Transformers (ViT). In ViT, an image is typically divided into a sequence of fixed-size patches. Each patch is then linearly embedded, and positional encodings are added. This sequence of patch embeddings is then fed into a Transformer encoder, where Self-Attention layers process these patches, allowing the model to learn relationships between different parts of the image, similar to how it learns relationships between words in a sentence.

9.  **What are some disadvantages or limitations of Self-Attention?**
    *   **Answer:**
        *   **Quadratic Complexity:** The $O(N^2)$ computational and memory complexity can be prohibitive for very long sequences.
        *   **Lack of Inductive Bias:** It lacks the inherent inductive bias for sequential data that RNNs have, requiring positional encodings.
        *   **Data Hungry:** Models relying heavily on Self-Attention (like Transformers) often require very large datasets to train effectively due to their high parameter count.

10. **How does Self-Attention contribute to the "contextualization" of embeddings?**
    *   **Answer:** Self-Attention allows each word's representation to be a weighted sum of all other words' Value vectors in the sequence. The weights are dynamically calculated based on the relevance of each word to the current word. This means that the output embedding for a word is not static but is influenced by its entire context in the current input. For example, the word "bank" would have a different contextualized embedding in "river bank" versus "money bank" because the attention mechanism would weigh "river" or "money" more heavily, respectively, when forming the representation of "bank."

    A) $O(N)$
    B) $O(N \log N)$
    C) $O(N^2)$
    D) $O(N^3)$

5.  What mechanism is typically added to Self-Attention to incorporate information about the order of elements in a sequence?
    A) Batch Normalization
    B) Dropout
    C) Positional Encodings
    D) Residual Connections

### Answer Key

1.  **B) It can process sequences in parallel, speeding up training.**
    *   **Explanation:** Self-Attention allows for parallel computation of attention weights for all elements, which is a major speed advantage over the sequential nature of RNNs.

2.  **C) It represents "what information I can offer."**
    *   **Explanation:** The Key vector is what other Query vectors compare themselves against to determine relevance.

3.  **C) To prevent the softmax function from having extremely small gradients due to large input values.**
    *   **Explanation:** Scaling by $\sqrt{d_k}$ helps stabilize training by keeping the inputs to the softmax function in a more manageable range, preventing vanishing gradients.

4.  **C) $O(N^2)$**
    *   **Explanation:** The calculation of the attention scores ($QK^T$) involves matrix multiplication that results in an $N \times N$ matrix, leading to a quadratic complexity with respect to the sequence length $N$.

5.  **C) Positional Encodings**
    *   **Explanation:** Self-Attention itself is permutation-invariant. Positional Encodings are added to the input embeddings to inject information about the absolute or relative position of each element in the sequence.
